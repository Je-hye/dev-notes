# gesture-slides 회고

## 개요

카메라로 손동작을 인식해 슬라이드를 제어하는 브라우저 전용 프레젠테이션 데모.
MediaPipe Hands JS + 단일 HTML 파일, 백엔드 없음. hackathon-mode 스킬로 약 1시간 내 완성.

> **이 프로젝트는 `hackathon-mode` 커스텀 스킬을 실제 빌드에 적용해 검증하기 위한 테스트였다.**
> 스킬의 SCOPE → STACK → PLAN → EXECUTE → DEMO CHECK 워크플로우를 그대로 따라가며
> 스킬이 실제 데드라인 상황에서 유효하게 동작하는지 확인했다.

## 목표

- 손 제스처(☝️ 레이저, ✌️ 이전, ✋ 다음)로 슬라이드 제어
- 설치 없이 브라우저에서 바로 실행 가능한 단일 파일
- 3시간 마감 내 완성

## 기술 스택

| 항목 | 선택 |
|------|------|
| 손 감지 | MediaPipe Hands JS (CDN) |
| UI | HTML + CSS + Vanilla JS |
| 배포 형태 | 단일 `.html` 파일 |
| 백엔드 | 없음 |

## 잘 된 것

- **단일 HTML 선택**이 옳았다. 빌드 없이 `open index.html`로 즉시 실행 가능했고 데모 흐름 전체가 파일 하나에 담겼다.
- **MediaPipe CDN**이 안정적으로 동작했다. 오프라인 환경에서는 CDN 의존성이 리스크지만 해커톤 데모 범위에서는 적합한 선택이었다.
- **Hold 타이머(900ms) + 쿨다운(1300ms)** 조합이 오인식을 효과적으로 줄였다.
- hackathon-mode 스킬의 SCOPE 단계가 must_have를 빠르게 고정시켜 구현에 집중할 수 있었다.

## 어려웠던 것

- **엄지 감지**: 수직(y) 비교 방식은 손 기울기에 민감하다. 엄지가 펴진 상태인데도 접혀 있다고 판정될 수 있어, next 제스처(4개 이상 펴기) 판정에서 엄지를 의도적으로 제외하는 방식이 더 안정적일 수 있다.
- **레이저 미러링 오프셋**: 웹캠은 `scaleX(-1)`로 미러링되는데, 슬라이드 컨테이너 좌표로 변환할 때 `(1 - tip.x)`로 반전시켜야 맞다.

## hackathon-mode 스킬 검증 결과

| 단계 | 평가 |
|------|------|
| SCOPE | 유효 — must_have 3개로 즉시 고정됨 |
| STACK | 유효 — 단일 HTML 선택 기준이 명확했음 |
| PLAN | 유효 — demo_critical 플래그로 우선순위 명확 |
| EXECUTE | 유효 — 순차 실행으로 충분, 병렬화 불필요했음 |
| DEMO CHECK | 부분 유효 — 오프라인 체크 항목은 CDN 사용으로 패스 불가 |

전반적으로 스킬이 실제 빌드 상황에서 잘 동작했다.
DEMO CHECK의 "네트워크 없이 동작하는가" 항목은 CDN 기반 프로젝트에는 맞지 않아 향후 스킬에 예외 조건 추가를 고려할 것.

## 기술 부채 (hackathon-debt)

- 엄지 감지 로직 개선 (수직 비교 → 각도 또는 좌우 비교)
- CDN 오프라인 대응 (assets 로컬 복사)
- 슬라이드 내용 외부 데이터 (JSON/마크다운)로 분리

## 배운 것

- MediaPipe Hands의 21개 랜드마크 인덱스 구조 (tip: 4/8/12/16/20, pip: 3/6/10/14/18)
- 브라우저 단일 파일 데모의 속도 이점 — 해커톤에서 빌드 파이프라인은 시간 낭비다
- hackathon-mode 스킬이 실제로 스코프 컷과 스택 결정을 빠르게 하는 데 유효하다는 것 확인
