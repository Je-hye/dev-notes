# 반도체 공정의 단계별 불량 원인 분석을 통한 불량률 최소화

## Overview
- **기간:** 2024.07 ~ 2024.08 (2주)
- **규모:** 3인 팀 프로젝트
- **역할:** 배경 분석, 목표 설정, 리스크 분석, 상관관계 규명, 시각화, 피처 선별
- **목적:** 반도체 공정 데이터 분석으로 수율 75% → 95% 향상 방안 도출
- **수상:** Python 활용 실무 데이터 분석 프로젝트 최우수상 (데이터스테이션 아카데미)

## Tech Stack
- **Language:** Python
- **라이브러리:** Pandas, NumPy
- **분석 방법론:** 가설 검정(T-test), 상관계수 분석, 데이터 핸들링
- **시각화:** Scatter Plot, Heat Map
- **모델:** Random Forest Regressor

## Architecture
**분석 파이프라인:** 배경 분석 → 목표 설정 → 리스크 분석 → 상관관계 규명 → 모델 피처 선별 → 시각화

**공정별 불량 위험도 분류:**
- 높음: 포토공정(이물질/정렬 오류), 식각공정(과식각/부족식각), 증착공정(두께 균일성)
- 중간: 배선/산화/이온 주입 공정
- 낮음: 품질 검사, 포장 공정

**핵심 발견:**
- 산화 공정: Thickness < 700nm → 다음 공정 진행 불가 불량 발생
- 포토 공정: Line_CD < 25nm 또는 > 55nm → 불량 임계 조건

| Decision | Options Considered | Choice | Reason |
|----------|--------------------|--------|--------|
| 불량 위험도 가중치 | 균등 가중치 vs 공정별 차등 | 차등 (고위험 20%, 중간 10%, 저위험 5%) | 공정별 영향도 차이를 반영해 우선순위 있는 개선 전략 수립 |
| 핵심 변수 선별 방법 | 전체 변수 투입 vs 상관관계 기반 선별 | 상관관계 분석 후 선별 | Random Forest 모델 성능 향상 + 해석 가능성 확보 |

## Challenges
**공정 데이터의 불량 임계값 특정**
- 다양한 공정 변수 중 어떤 조건에서 불량이 발생하는지 수치로 특정하는 것이 핵심
- 각 변수의 분포와 이상치를 분석하고 T-test로 통계적 유의성 검증

## Learnings & Retrospective
- 데이터 분석은 기술보다 "어떤 질문을 던지는가"가 핵심임을 배움
- 경제적 손실 수치(약 9조 원)로 문제의 비즈니스 임팩트를 먼저 정량화하는 접근이 설득력 있었음
- 비전공(고분자공학) 시절 처음 접한 데이터 분석 프로젝트 — 전과 결정의 계기
